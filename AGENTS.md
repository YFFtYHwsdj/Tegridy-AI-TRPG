# AI 跑团系统

## 项目

AI 主持的桌面角色扮演游戏。核心规则以 PBTA 为骨架，融合《异景 / :Otherscape》机制。
玩家用自然语言描述行动，所有规则翻译工作由多 Agent 系统在幕后完成。
**玩家只说人话。规则藏在 Agent 背后。**

## 技术

- Python 项目，本地 `.venv` 管理依赖，本地venv版本为3.11，请优先使用本地环境。
- LLM API：DeepSeek（OpenAI-compatible 格式）
- `prompt, rules and conpects/` 目录下为 Agent prompt 参考模板，包含大量规则信息，非运行时直接加载
- 代码质量：使用 [ruff](https://docs.astral.sh/ruff/) 进行 lint 和格式化，配置在 `pyproject.toml` 的 `[tool.ruff]` 段
  - `ruff check .` — 检查代码质量
  - `ruff check --fix .` — 自动修复可修复的问题
  - `ruff format .` — 格式化代码

## 核心设计原则

1. **叙事与规则分离** — 玩家侧零规则暴露。意图解析、标签匹配、行动拆分、效果分配、后果兑现全部在 Agent 流水线内部完成，最终只输出沉浸式叙事文本。
2. **多 Agent 协作** — 系统由裁判、效果推演、后果、叙述者、节奏等多个专门 Agent 组成，各司其职，接力完成从规则裁判到叙事渲染的完整流水线。
3. **效果推演优先于效果计算** — 不由玩家花费力量购买效果。效果推演 Agent 作为因果模拟器，思考"以这个角色的能力，在这个环境中做这件事，实际上会造成什么"，然后翻译为效果类型和等级。

## 关键编码约定

1. **可判定逻辑用代码，需理解判断用 LLM** — 力量计算、掷骰解析、状态叠加溢出、极限达成检测用纯 Python（确定性、零 token 消耗）；标签适用性判断、意图解析、效果推演、叙事生成由 LLM Agent 处理。
2. **Agent 间传递"分析便签"而非"表单"** — Agent 输出是带推理过程的自然语言分析。下一个 Agent 读到前一个的完整思考过程，做出更连贯的接力判断。固定 JSON 字段只在力量值、掷骰结果等真正需要的地方使用。
3. **场景作为上下文单元** — 游戏按场景推进。场景结束时压缩 Agent 将对话历史压缩为结构化摘要，下一场景以前情提要启动，避免上下文无限膨胀。
4. **每次修改代码后必须运行 ruff** — 任何 `.py` 文件修改完成后，必须依次执行 `ruff check .` 和 `ruff format .`，确保代码质量零告警。这是强制步骤，不可跳过。
