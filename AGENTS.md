# AI 跑团系统 · 设计概要

## 项目目的

构建一个 AI 主持的桌面角色扮演游戏系统。核心规则以 PBTA（Powered by the Apocalypse）为骨架，融合迷雾引擎（尤其是最新作《异景 / :Otherscape》）的机制。

玩家通过自然语言描述角色的行动和意图，AI 系统完成全部规则翻译工作——判定标签适用性、计算力量值、执行掷骰、分配效果、兑现后果——最终将一切翻译回沉浸式的叙事文本。

**玩家只说人话。规则藏在 Agent 背后。**

***

## 开发环境

- **语言**：Python
- **虚拟环境**：使用本地 `.venv` 管理依赖
- **LLM API**：DeepSeek（OpenAI-compatible 格式）

***

## 设计哲学

### 一、叙事与规则分离

玩家侧不需要知道任何规则的存在。玩家的输入是纯粹的叙事意图——"我把守卫摔到墙上"。所有机械操作——意图解析、标签匹配、行动拆分、效果分配、后果兑现——由 AI 系统在内部完成，最后返回的是一段流畅的叙事文本。

效果系统的全部复杂度完整保留（攻击、影响、干扰、削弱、赋予、增强、恢复、推进、挫败、发现、创造、额外壮举），但隐藏在多 Agent 协作的流水线之中。

### 二、多 Agent 分工

单一 Agent 无法胜任从规则裁判到叙事渲染的全部工作。系统由多个专门 Agent 组成，各司其职，例如：

- **裁判类 Agent**：解析玩家意图、匹配适用标签、判断行动是否拆分、计算力量
- **效果推演 Agent**：作为因果模拟器——不是做数值优化，而是推演"在这个世界设定下，这个角色的这个行动实际会造成什么"
- **后果 Agent**：在部分成功或失败时，根据挑战的威胁列表选择并施加后果
- **叙述者 Agent**：将所有结构化结果翻译为沉浸式叙事文本，但只做翻译，不做判断
- **节奏 Agent**：控制 PBTA 的建立—行动—后果循环，决定聚光灯投向谁、场景是否结束、何时建立威胁、弱点标签是否援引
- **创作 Agent**：在需要新挑战、新物品、新场景时介入，承担世界构建工作
- **NPC Agent**：为重要角色维护独立的上下文和内部状态机，能主动采取行动，而非被动应答
- **上下文压缩 Agent**：在场景边界将对话历史压缩为结构化摘要，供后续场景使用

### 三、效果推演优先于效果计算

传统的"玩家花费力量购买效果"模式不适合 AI 主持。在本系统中，效果推演 Agent 收到玩家的意图描述和掷骰结果后，思考的是：

> 以这个角色的能力，在这个场景环境中，做这件事，实际上会造成什么？

然后选择合适的效果类型和等级来翻译这个推演结果。这不是在填表——是在模拟因果。

### 四、场景作为上下文单元

游戏的叙事天然按场景推进（起始场景 → 后续场景 → 终章）。系统以此为上下文切换的天然边界：

- 游戏进行中，当前场景的所有状态保持在活跃上下文中
- 场景结束时，上下文压缩 Agent 生成结构化摘要（关键事件、状态变化、遗留线索）
- 下一个场景以该摘要作为前情提要启动
- 更早的历史作为按需加载的世界状态

这种设计避免了无限膨胀的对话历史，同时保持了叙事连贯性。

### 五、Agent 间传递"分析便签"而非"表单"

Agent 之间的数据传递不采用僵硬的 JSON schema。每个 Agent 的输出是一段**带推理过程的自然语言分析**，像是经验丰富的 GM 在笔记本上的快速备注。下一个 Agent 可以读到前一个 Agent 的完整思考过程，做出更连贯的接力判断。

固定字段只在真正需要的地方使用（如力量值、掷骰结果），其余保持自然语言的灵活性。

### 六、可判定逻辑用代码，需理解判断用 LLM

力量计算、掷骰解析、状态叠加溢出、极限达成检测——这些是确定性规则，用纯 Python 代码实现，不消耗 token，结果绝对一致。

标签适用性判断、意图解析、效果推演、叙事生成——这些需要语义理解和创造性，由 LLM Agent 处理。

## 核心游戏循环

每轮游戏遵循以下流程：

1. **节奏检查**：场景是否该结束？是否该建立威胁？聚光灯给谁？弱点是否触发？
2. **玩家输入**：自然语言描述行动和意图，可自然嵌入想要使用的标签名
3. **裁判流水线**：意图解析 → 标签匹配 → 行动拆分判定 → 力量计算 → 掷骰
4. **效果推演**：根据行动、能力、环境推演实际效果，翻译为效果类型和等级
5. **后果兑现**：如果掷骰产生了后果，从挑战的威胁列表中选择并施加
6. **状态更新**：应用所有效果，处理溢出，检查极限达成
7. **叙事输出**：将以上一切翻译为沉浸式叙事文本，呈现给玩家

## 规则简化与适配

异景原版规则中，以下部分针对 AI 主持做了调整：

- **效果分配**：不由玩家手动花费力量购买，由效果推演 Agent 自动匹配意图
- **弱点援引**：不再依赖玩家主动声明，由节奏 Agent 根据情境自动检测
- **威胁建立与兑现**：节奏 Agent 按触发条件自动选择和执行，减少 MC 自由裁量
- **行动拆分**：有明确的判据（时序分离、目标分离、效果类型跨度过大）

其余核心机制完整保留：2d6 + 力量、10+/7-9/6- 判定、标签系统、挑战的极限和进度极限、状态叠加与溢出。

***

## 角色创建

通过对话式向导完成。AI 按主题问卷依次提问，玩家用自然语言回答。AI 解析答案，生成带语义标注的结构化标签（关键词、适用行动类型、燃尽效果）。最后整合为完整的人物卡。

提供两种模式：快速创建（预设主题套组）和自由创建（逐主题回答问卷）。
